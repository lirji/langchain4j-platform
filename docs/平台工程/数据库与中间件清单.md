# 数据库与中间件清单（账号 / 密码 / 端口）

> 本文汇总 langchain4j-platform 用到的**所有数据存储、消息中间件、AI 网关**及其连接信息（主机、端口、库名、账号、口令）。
>
> 信息来源：`deploy/docker-compose*.yml`、各服务 `application.yml`、`deploy/helm/platform/values.yaml`、`analytics-service/.../db/nl2sql-demo.sql`。
>
> ⚠️ **安全提示**：以下账号/口令均为**本地开发默认值**（如 `root/root`、`demo12345`、`sk-litellm-master`、`dev-only-*-secret-*`）。**生产环境必须全部替换**——Helm 里已用 `change-me` 占位并建议接 External Secrets Operator / Vault（见 `values.yaml` 的 `secrets.*`）。切勿把真实口令提交进仓库。

## 二、MySQL（核心关系数据库）

- **镜像**：`mysql:8.4`
- **宿主端口**：`13307`（`deploy/.env`/`start-all.sh` 默认，避开本机 Apollo；裸 `docker compose up` 回落 `3306`，`MYSQL_HOST_PORT` 覆盖，容器内恒为 3306）
- **容器内主机名**：`mysql`（k8s Service 名同名，跨服务硬编码主机名零改动）
- **root 账号**：`root` / `root`（`MYSQL_ROOT_PASSWORD: root`）
- **初始库**：`platform`（`MYSQL_DATABASE`，占位初始化）
- **数据卷**：`mysql-data`（命名卷持久化）
- 各库均带 `createDatabaseIfNotExist=true`，首次连接自动建库；**表结构靠各 `Jdbc*Store` 里的 `CREATE TABLE IF NOT EXISTS` 演进**（无 Flyway/Liquibase）。

### 2.1 业务库一览

| 数据库 | 所属服务 | 连接账号 | 密码 | JDBC URL（compose 内） | 用途 |
|--------|----------|----------|------|------------------------|------|
| `flowable` | workflow-service (:8082) | `root` | `root` | `jdbc:mysql://mysql:3306/flowable?...serverTimezone=UTC` | Flowable BPMN 退款审批引擎自管表 + outbox |
| `nl2sql_demo` | analytics-service (:8083) | `root`（管理）/ `nl2sql_ro`（只读） | `root` / `nl2sql_ro` | `jdbc:mysql://mysql:3306/nl2sql_demo?...` | NL2SQL 演示业务数据 |
| `knowledge_graph` | knowledge-service (:8084) | `root` | `root` | `jdbc:mysql://mysql:3306/knowledge_graph?...` | GraphRAG 三元组 `RAG_GRAPH_TRIPLE`（`RAG_GRAPH_STORE=jdbc` 时） |
| `async_task` | async-task-service (:8086) | `root` | `root` | `jdbc:mysql://mysql:3306/async_task?...` | 异步任务中心/租约/webhook outbox（`ASYNC_TASK_STORE=jdbc` 时） |
| `auth` | auth-service (:8092) | `root` | `root` | `jdbc:mysql://mysql:3306/auth?...` | 登录 + RBAC（`AUTH_STORE=jdbc` 时）：`USERS`、`AUTH_SESSION`、`ROLES` + 关系表 `USER_ROLE`/`ROLE_SCOPE`（权威，CSV 列影子双写） |

| `platform` | — | `root` | `root` | — | MySQL 初始化建的占位库 |

### 2.2 NL2SQL 只读账号（analytics-service）

analytics 用**独立只读账号**执行模型生成的 SQL（防写入），由种子脚本 `db/nl2sql-demo.sql` 创建：

```sql
CREATE USER IF NOT EXISTS 'nl2sql_ro'@'%' IDENTIFIED BY 'nl2sql_ro';
GRANT SELECT ON nl2sql_demo.* TO 'nl2sql_ro'@'%';
```

- **只读账号**：`nl2sql_ro` / `nl2sql_ro`（仅 `SELECT nl2sql_demo.*`）
- **管理账号**（建库/建表/灌种子）：`root` / `root`
- 覆盖用环境变量：`NL2SQL_DB_ADMIN_USER/PASSWORD`、`NL2SQL_DB_READONLY_USER/PASSWORD`

### 2.3 各服务默认口令差异（本地直跑 vs compose）

|----------|-------------|---------------------------|-----------|
| `WORKFLOW_DB_PASSWORD` | `root` | `root` | `change-me` |
| `NL2SQL_DB_ADMIN_PASSWORD` | `root` | `root` | `change-me` |
| `ASYNC_TASK_DB_PASSWORD` | `root` | **空** | `change-me` |
| `RAG_GRAPH_DB_PASSWORD` | `root` | **空** | `change-me` |
| `AUTH_DB_PASSWORD` | `root` | **空** | — |

> 注意：不带 compose 时，`async_task` / `knowledge_graph` / `auth` 的 yml 兜底口令为**空字符串**——本地裸跑 MySQL 若 root 有密码需显式传环境变量。

## 三、Redis（KV / 缓存 / 会话）

- **镜像**：`redis:7-alpine`，启动参数 `--appendonly yes`（AOF 持久化）
- **端口**：`6379`
- **账号 / 密码**：**无认证**
- **容器内主机名**：`redis`（`SPRING_DATA_REDIS_HOST` / `SPRING_DATA_REDIS_PORT`）
- **数据卷**：`redis-data`
- **用途**：限流（`RATE_LIMIT_STORE=redis`）、令牌预算（`TOKEN_BUDGET_STORE=redis`）、成本、L1 语义缓存、RAG 文档注册表（`RAG_REGISTRY_STORE=redis`）、多轮 Chat Memory（`CONVERSATION_MEMORY_STORE=redis`）
- **使用方**：edge-gateway、conversation、knowledge、agent、workflow、analytics、channel、vision 等

## 四、Kafka（消息中间件）

- **镜像**：`apache/kafka:3.8.0`（KRaft 模式，无 ZooKeeper）
- **端口**：`9092`（PLAINTEXT，broker 对外）、`9093`（controller，仅容器内）
- **账号 / 密码**：**无认证**（`PLAINTEXT`）
- **容器内主机名**：`kafka`，`KAFKA_BOOTSTRAP_SERVERS: kafka:9092`
- **默认 topic**：`platform.channel.events`（`CHANNEL_EVENTS_TOPIC`）
- **使用方**：channel-service（渠道出入站事件，`CHANNEL_EVENTS_ENABLED=true` 时）、platform-eventbus Kafka 变体

## 六、Elasticsearch + Kibana（默认开启，ES 全文混排）

**已并入基础 `deploy/docker-compose.yml` 默认开启**（`RAG_ES_ENABLED` 代码默认也已翻为 `true`）。
`docker compose up` / `start-local.sh` / `start-all.sh` / `start-dev.sh` 均自动拉起 ES + Kibana 并启用 ES 混排。
要退回零依赖：`export RAG_ES_ENABLED=false`（并按需 `RAG_EMBEDDING_PROVIDER=hash`）。
> 无 ES 可达也不影响启动：ES 客户端惰性连接、索引惰性创建、查询期 ES 源失败自动降级（其余源照常）。

### Elasticsearch
- **镜像**：自建 `deploy/es`（Elasticsearch 8.15.x + `analysis-smartcn` 中文分析器）
- **端口**：`9200`
- **账号 / 密码**：**无认证**（`xpack.security.enabled=false`；`RAG_ES_USERNAME/PASSWORD/API_KEY` 默认全空）
- **容器内地址**：`http://elasticsearch:9200`（`RAG_ES_URIS`）
- **索引**：`knowledge_segments_text`（`RAG_ES_INDEX_NAME`），分析器 `smartcn`
- **数据卷**：`es-data`
- **融合策略**：`RAG_FUSION_STRATEGY=rrf`（RRF，免疫 BM25/余弦量纲差）

### Kibana
- **镜像**：`docker.elastic.co/kibana/kibana:8.15.3`
- **端口**：`5601`（http://localhost:5601 → Dev Tools）
- **账号 / 密码**：无
