# 能力展示与试用控制台 —— 功能与能力说明

> 本文面向使用者/评审者，说明**前端页面上能做什么**以及**页面提供了哪些平台能力**。
> 架构、部署与配置见 [能力展示控制台](能力展示控制台.md)；工程自述见 `capability-showcase-frontend/README.md`；模块拆分依据见 [能力前端模块拆分建议](能力前端模块拆分建议.md)。

`capability-showcase-frontend` 是一个 **Vue3 + TS + Vite** 的独立静态 SPA，把 langchain4j-platform 的后端能力**统一收进一个可浏览、可直接试用的控制台**。浏览器以 direct mode 带凭证跨域直调 `edge-gateway`，无需任何展示后端。

- **覆盖范围**：**9 个功能模块、82 条能力**，端点/入参对照真实 controller 与 `platform-protocol` DTO 核验，未臆造。
- **能力目录唯一事实源**：`capabilities.yml`（构建期生成 `catalog.json`）—— 每条能力的方法/路径/参数/所需 scope/feature-flag/风险级别/可用状态都在这里声明。

## 一、页面整体：一处试用全部能力

控制台是「**左侧三级导航 + 右侧能力工作台**」的单页结构。左边按语义组织能力、右边填表即发起真实调用并可视化响应。

### 1.1 左侧导航（三级 + 彩色编码 + 五态状态点）

按「**语义分组 → 模块 → 能力**」三级组织，分组彩色编码，一眼分辨能力类别与可用性：

| 分组 | 颜色 | 包含模块 |
|---|---|---|
| 对话与检索 | 蓝 | Chat、RAG |
| 智能体与编排 | 紫 | Agent、Async、Workflow、Analytics |
| 多模态 | 琥珀 | Multimodal |
| 平台工程与互操作 | 青 | Interop & Eval、Channel |

- **五态状态点**：每条能力行显示 HTTP 方法 + 状态点（颜色 + 形状双编码，附文字图例，不单靠颜色区分）。五态含义见 [第四节](#四能力五态状态语义)。
- **筛选**：顶部筛选框按标题 / id / 路径 / 描述 / 标签子串匹配；`/` 键聚焦；无结果显示空态与清除入口。
- **折叠/展开**：可折叠分组、展开模块下的能力；桌面可折叠整条侧栏；移动端为抽屉（`Esc`/点遮罩关闭，关闭后焦点归还，隐藏侧栏 `inert` 不被 Tab/读屏）。
- **收藏**：能力可收藏，置顶「收藏」虚拟分组。
- **密度**：紧凑/舒适切换。折叠态、收藏、密度均持久化到本地。

### 1.2 右侧工作台（数据驱动 + 领域专用双底座）

- **通用运行器（`CapabilityRunner`）**：绝大多数能力由 `capabilities.yml` 声明**自动生成动态表单** —— query/body/path 参数、文本域、下拉、布尔、文件上传、原样 JSON 字段（承载 eval cases、dag tasks、mcp arguments 等复杂嵌套体）都由声明驱动。
- **领域专用工作台**：`workflow / multimodal / interop-eval / channel / rag / agent / tasks / analytics` 有定制视图（并排双区、时间线、指标卡等），但**复用同一个运行器与执行闸门，不绕过安全裁决**。
- **响应可视化**：JSON 折叠视图；SSE 逐 token 控制台（断流带 `Last-Event-ID` 续订）；表格化行集、指标卡（Recall@k/MRR/Hit@k）、图像缩略图/音频播放器预览；探测不到结构则诚实回退到通用 `ResponseViewer`，不臆造字段。

### 1.3 全局能力（跨所有模块）

- **cURL 导出**：任一能力可一键复制等价 `curl`（凭证脱敏），把页面操作变成可复现命令。
- **请求历史**（`⌘J`）：记录发起过的调用，可回看/重放。
- **命令面板**（`⌘K`）：模糊搜索并直达任意能力。
- **快捷键**：`⌘K` 命令面板、`⌘J` 历史、`⌘/` 快捷键帮助、`⌘⇧L` 主题、`⌘⇧D` 密度、`/` 聚焦筛选。
- **主题**：亮/暗色主题切换。
- **权限预判**：登录态下按账号 scope 预判某能力是否可执行（最终以服务端鉴权为准）。
- **凭证安全**：API Key / token 仅存浏览器内存，绝不落 URL/localStorage/日志；导出的 curl 亦脱敏。

## 二、九大功能模块与提供的能力（82 条）

## 三、认证与访问方式

页面提供**三种进入方式**，由构建期 `VITE_AUTH_MODE` 三态灰度，可秒级回滚：

1. **API Key 直连**（`apikey`，默认）：顶栏填网关认可的 `X-Api-Key`，direct mode 跨域直调。加载能力目录不需要 Key，试用业务能力需要。
2. **账号密码登录**（auth-service）：登录得会话 JWT（仅内存，Bearer）+ httpOnly 刷新 cookie 静默续期；边缘用会话 Bearer 换发内部 JWT，下游零改动。支持自助注册（遵从后端 `/auth/public-config` 的注册开关与密码策略）。**顶栏填 Key 时覆盖登录会话**（高对比警告提示）。
3. **Casdoor OIDC 单点登录**（`oidc` / `dual`，构建期灰度）：Authorization Code + PKCE，业务请求带 `Authorization: Bearer <casdoor-token>`，网关验签换发形状不变的内部 JWT，使 `Casdoor sub = 内部 JWT uid = TenantContext.userId = SpiceDB user id` 四者一致。多租户走**方案C**（Shared Application + 登录前选组织，每租户以 `<base>-org-<tenant>` 客户端登录）。含近失效静默续期、会话过期模态（不丢 deep-link）、`end_session` + `BroadcastChannel` 多标签登出同步。

> 平台管理模块（用户/角色/租户/用户组管理）已从本控制台移除，身份与授权统一由外部 IAM（Casdoor SSO + auth-platform）承担，本前端只消费登录会话与 scopes。

## 四、能力五态（状态语义）

每条能力标注单一状态，导航状态点与运行器执行闸门共用这套语义 —— **诚实呈现「能不能跑、为什么不能跑」**，不让用户对着一个注定 403/404 的按钮反复试：

| 状态 | 标记 | 含义 |
|---|---|---|
| **就绪** | ● | 默认可执行 |
| **就绪·降级** | ◐ | 可执行，但为内存/确定性降级实现（非真实语义/生产依赖），如 `/rag/query` 默认 HashEmbedding |
| **需授权** | ◆ | 需特定 scope（如 ingest/approve），否则 403 |
| **未启用** | ○ | 需开启对应 feature flag 才注册，默认不可执行 |
| **已锁定** | ■ | 破坏性写/删，默认仅展示不执行（如 `DELETE /workflow/data`、`POST /channel/messages`）|

> kill switch 语义：即便前端开关置 true，能力是否真正可用仍由服务端 scope / 运行时配置决定；破坏性能力即使有权限也需在页面显式解除锁定。

## 五、请求三形态

页面统一处理三种请求/响应形态，无需切工具：

- **JSON**：结构化请求体 + 折叠 JSON 响应视图。
- **multipart 上传**：文件字段带拖拽区与图像/音频预览。
- **SSE 流式**：浏览器 `fetch` + `ReadableStream` 逐 token 消费，SSE 控制台展示；断流带 `Last-Event-ID` 续订。个别语音能力为 `multipart-sse`，运行器自动切换。
