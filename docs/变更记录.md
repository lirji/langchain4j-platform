本文汇总 **2026-07** 的方案级（架构 / 能力 / 默认值 / 使用方式）变更，便于快速了解「最新版本相对早期文档变了什么」。细粒度接入说明见各专题文档；本文只做导航式摘要，按主题倒序排列。

## 0. 默认值变化速览（最重要）

2026-07 有两处**默认行为翻转**，是文档最容易过时的地方：

### (a) 身份鉴权默认 = Casdoor ONLY（严格）

| 开关 | 旧默认 | 新默认 | 说明 |
| --- | --- | --- | --- |
| `edge.casdoor.enabled` (`EDGE_CASDOOR_ENABLED`) | `false`（关） | **`true`（开）** | edge 验 Casdoor JWT 换发内部 JWT |
| `edge.casdoor.mode` (`EDGE_CASDOOR_MODE`) | `dual` | **`only`（严格）** | 非 open-path 无有效 Casdoor token → 401，tenant 恒取 `owner`；`dual` 降级为灰度回滚窗口 |
| 前端 `VITE_AUTH_MODE` | `apikey` | 源码仍 `apikey`、**整栈构建期烘焙 `oidc`**（`SHOWCASE_AUTH_MODE`） | 控制台默认走 Casdoor OIDC 登录 |

影响：**本地/真机整栈必须先起外部 auth 栈（Casdoor + auth-platform）**，否则登录/业务请求 401；`bash deploy/start-all.sh` 会检测并提示。应急退回旧凭据：`EDGE_CASDOOR_MODE=dual`（Casdoor 验不过时回落会话 Bearer / api-key）。详见 [公网化 OIDC 改造方案](平台工程/公网化-OIDC-改造方案.md)、[登录、RBAC 与公共知识库](平台工程/rbac-and-public-kb.md)。

### (b) 各服务增强开关默认全开（2026-07-17）

各服务 `application.yml` 里大量 `${ENV:false}` 翻为 `:true`。**现默认开**：

- 检索/RAG：`RAG_HYBRID_ENABLED`、`RAG_ES_ENABLED`(+index/query)、`RAG_RERANK_ENABLED`、`RAG_QUERY_EXPANSION_ENABLED`、`RAG_CONTEXTUAL_ENABLED`、`RAG_GRAPH_ENABLED`（`store=jdbc`）、`RAG_MULTIMODAL_ENABLED`、`RAG_PUBLIC_ENABLED`、`RAG_CACHE_INVALIDATION_ENABLED`
- 对话：`CONVERSATION_RAG_ENABLED`、`CONVERSATION_HISTORY_AWARE_ENABLED`、`CONVERSATION_MEMORY_PROFILE_ENABLED`、`CONVERSATION_GROUNDING_ENABLED`、`CONVERSATION_ROUTER_ENABLED`、`CONVERSATION_VISION_ENABLED`、`CONVERSATION_GUARDRAIL_INJECTION_ENABLED`、`CONVERSATION_GUARDRAIL_PII_ENABLED`、`CONVERSATION_SEMANTIC_CACHE_ENABLED`、`CHAT_CASCADE_ENABLED`
- Agent：`AGENT_ENABLED`、`AGENT_VISION_ENABLED`、`AGENT_ANALYTICS_ENABLED`、`AGENT_WORKFLOW_ENABLED`、`AGENT_ORDER_ENABLED`、`AGENT_ASYNC_EXTERNAL_ENABLED`、`AGENT_DAG_REPLAN_ENABLED`
- 其它：`NL2SQL_ENABLED`、`WORKFLOW_ENABLED`、`INTEROP_DISCOVERY_ENABLED`、`EVAL_JUDGE_ENABLED`、`EVAL_EMBEDDING_ENABLED`、`VISION_ENABLED`

**仍默认关**（需显式开）：`CONVERSATION_MCP_ENABLED`、`AGENT_CODE_EXEC_ENABLED`、`AGENT_MCP_ENABLED`、`AGENT_BROWSER_ENABLED`、`VOICE_ENABLED`、`FEISHU_*` / `DINGTALK_*`、`CHANNEL_EVENTS_ENABLED`、`RAG_AUTHZ_MODE`(=`disabled`)、`AUTH_RBAC_ENABLED`（yml 层；compose 整栈开）、`AUTH_REGISTRATION_ENABLED`、`cost`。

影响：**裸跑单个服务已不再零依赖**（GraphRAG 走 jdbc/MySQL、向量库走 qdrant、ES 走 Elasticsearch、多模态需 embedding 后端、vision 需 `VISION_MODEL` 否则 fail-fast）；真正零依赖单跑需显式退回（`RAG_EMBEDDING_PROVIDER=hash` + `RAG_ES_ENABLED=false` + `RAG_VECTOR_STORE_PROVIDER=in-memory` 等）。单元测试是纯 POJO、不启动 Spring，**零外部依赖恒成立**。

## 1. 身份与授权（最大架构变更）

- **自建账号密码登录体系** —— 新增 `auth-service`（:8092）：账号密码登录 → 会话 JWT（Bearer，内存，60min）+ httpOnly 刷新 cookie（7d）；edge `SessionBearerAuthFilter` 验签换发内部 JWT，下游零改动。→ [rbac-and-public-kb.md](平台工程/rbac-and-public-kb.md)
- **继承式 RBAC** —— 租户基础角色 + 用户组，三层向下继承；合成点 `EffectivePermissionResolver`；`/auth/admin/**` 管理面（If-Match 乐观锁）。yml 默认关，`deploy/docker-compose.yml` 整栈默认开（`AUTH_STORE=jdbc` + `AUTH_RBAC_ENABLED`/`ADMIN_WRITES`/`INHERITANCE=true`，bootstrap admin `alice`）。
- **Casdoor OIDC SSO（默认路径）** —— edge `CasdoorTokenExchangeFilter`（order -120）JWKS 验签换发内部 JWT，保持 `Casdoor sub = 内部 JWT uid = TenantContext.userId = SpiceDB user id` 一致。多租户 = **方案 C：Casdoor Shared Application + 选组织**，per-tenant client `<base>-org-<tenant>`，token `aud=<base>-org-<org>`。→ [公网化-OIDC-改造方案.md](平台工程/公网化-OIDC-改造方案.md)、[deploy/README-multitenant-login.md](../deploy/README-multitenant-login.md)
- **文档级 / 部门层级 ReBAC 授权** —— knowledge-service `app.rag.authz.mode`（`disabled`/`shadow`/`enforce`，**默认 `disabled`**，fail-closed）委托外部 **auth-platform-server（:8200，SpiceDB）** 做文档级判权 + 部门层级隔离；身份链 Casdoor group → `dept` claim → `TenantContext.department`。→ [rag-guide.md](对话与检索/rag-guide.md)、[rbac-and-public-kb.md](平台工程/rbac-and-public-kb.md)
- **前端 RBAC 管理控制台移除** —— 平台身份/授权统一外置到 Casdoor + auth-platform；能力展示前端不再随包 `/admin` 控制台。**后端 `/auth/admin/**` API 仍在**（受 `AUTH_RBAC_ENABLED` 门控），只是无随包 UI。

## 2. 检索 / RAG

- **ES 全文混排 + RRF 融合** —— Elasticsearch 真 BM25 全文并入四路 `RetrievalSource`（向量 / 内存关键词 / ES BM25 / 图谱），`HybridFusionService` 做多源 RRF 融合（`k=60`，ES 参与时自动选 RRF）；中文分词 smartcn / 可选 HanLP。→ [es-hybrid-rerank.md](对话与检索/es-hybrid-rerank.md)、[向量检索-ann与rrf.md](对话与检索/向量检索-ann与rrf.md)
- **共享/公共知识库** —— `__public__` 分区 + visibility 全链路 + `GET /rag/config` 运行时配置；写公共库需 `public-ingest` scope。
- GraphRAG、多模态 CLIP 图片检索、rerank/查询扩展/上下文分块 等增强默认开（见速览 (b)）。

## 3. Agent 与编排

- **数据分析智能体** `/agent/analyst/run`(+`/async`) —— DAG 编排「探表 → 取数 → 计算 → 解读」，复用 DAG 引擎（分层 / 并行 worker / synthesis / critic+replan）。
- **业务流程智能体（人在环）** `/agent/process/run` —— NL 发起/跟进退款审批；`AGENT_WORKFLOW_ENABLED` + `WORKFLOW_ENABLED` 现默认开，有副作用，需 workflow-service 可达。
- **`order_query` 动作** —— 接入新增 `order-service`（OrderClient SPI，Noop/Http 双实现）。
- ReAct 决策改走 `json_object` 模式，稳健性提升。→ [agent-guide.md](Agent编排/agent-guide.md)、[让Agent主动调接口.md](Agent编排/让Agent主动调接口.md)

## 4. 新增服务

- **`order-service`（:8093）** —— 按订单号只读查订单（状态/金额/客户/下单日期），裸 `JdbcTemplate` + MySQL 独立库 `order_service`、按 `tenant_id` 隔离；供 agent `order_query` 调用。**宿主机端口映射 8094**（宿主 8093 被展示前端占用）。
- **`voice-service`（:8091，默认关）** —— 语音闭环 `/voice/transcribe`·`/voice/chat`(+`/stream`)，ASR(whisper) → 对话大脑 → TTS。→ [voice-guide.md](多模态语音/voice-guide.md)

## 7. 部署与端口
